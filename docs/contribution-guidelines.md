# Contribution Guidelines

## Introduction
Thank you for considering contributing to the pfSense Walkthrough repository. Contributions are welcome and appreciated. The following guidelines outline the process for contributing to this project.

## How to Contribute

### Forking the Repository
1. **Fork the Repository**:
   - Navigate to the main repository page on GitHub.
   - Click the `Fork` button at the top right of the page to create a copy of the repository under your GitHub account.

2. **Clone the Forked Repository**:
   - Open your terminal and clone your forked repository to your local machine:
     ```bash
     git clone https://github.com/yourusername/pfSense-Walkthrough.git
     ```
   - Navigate into the cloned directory:
     ```bash
     cd pfSense-Walkthrough
     ```

### Creating a New Branch
1. **Create a New Branch**:
   - It is recommended to create a new branch for each contribution or bug fix:
     ```bash
     git checkout -b feature-or-bugfix-name
     ```

2. **Make Your Changes**:
   - Make the necessary changes or additions to the code or documentation.

### Committing Your Changes
1. **Stage Your Changes**:
   - Add the files you have changed:
     ```bash
     git add .
     ```

2. **Commit Your Changes**:
   - Write a clear and concise commit message:
     ```bash
     git commit -m "Brief description of the changes made"
     ```

### Pushing Your Changes
1. **Push to Your Forked Repository**:
   - Push your changes to your forked repository on GitHub:
     ```bash
     git push origin feature-or-bugfix-name
     ```

### Creating a Pull Request
1. **Navigate to the Main Repository**:
   - Go to the main repository page on GitHub.

2. **Create a Pull Request**:
   - Click the `New Pull Request` button.
   - Select the branch you have made changes to from your forked repository.
   - Provide a detailed description of your changes and any relevant information.
   - Click `Create Pull Request`.

## Code of Conduct

### Expected Behavior
- **Respect**: Show respect to all contributors and maintainers.
- **Inclusivity**: Foster an inclusive and welcoming environment.
- **Collaboration**: Collaborate openly and provide constructive feedback.

### Unacceptable Behavior
- **Harassment**: Harassment of any kind will not be tolerated.
- **Discrimination**: Discrimination based on race, gender, religion, or other personal characteristics is not allowed.
- **Disrespect**: Disrespectful behavior towards others is not acceptable.

### Reporting Issues
- **Contact**: If you experience or witness unacceptable behavior, please contact the repository maintainers directly through GitHub.

## Style Guide for Code Contributions

### Coding Conventions
- **Consistency**: Follow the existing coding style and conventions used in the project.
- **Comments**: Comment your code where necessary to explain complex logic.
- **Documentation**: Update or add documentation for any new features or changes.

### Documentation Contributions
- **Clarity**: Ensure that documentation is clear and easy to understand.
- **Detail**: Provide detailed explanations and examples where applicable.
- **Consistency**: Maintain a consistent style and format throughout the documentation.

## Conclusion
By following these guidelines, you can help maintain the quality and consistency of the pfSense Walkthrough repository. Your contributions are valuable and help improve this project for everyone.

Thank you for contributing!
